DELETE FROM에 WHERE 절 안쓰면 속하는 튜플들 모두 제거
CREATE TABLE ... AS
를 사용하면 해당 조건에 맞는 릴레이션이 있는 것처럼 보임
2가지 구현법
-뷰의 실체화 : 카피해서 보여주는 식 but 데이터의 불일치 발생 위험
-질의 수정 : 그 때그때 필요에 따라서 테이블을 구성해서 보여줌 but 할때마다 조인등의 연산으로 인해 성능이 떨어지지만, 데이터의 불일치는 발생ㄴ

정규화

릴레이션은 애트리뷰트로 구성

애트리뷰트를 그룹화를 잘못하게 되면 어떻게 되나?
-데이터 중복의 문제점 ( 불일치 - 갱신이상(Update anomaly))
=발생하는 경우에 따라 수정이상, 삽입이상, 삭제이상
=수정이상 : 둘 중 한놈만 수정되어 두 데이터간의 불일치가 발생하는 것
=삽입이상 : 그룹지어진 데이터 때문에 내가 원하는 일부의 데이터 뿐만 아니라 나머지 데이터도 넣어줘야만 하는 것
=삭제이상 : 그룹지어진 데이터 때문에 내가 원하는 일부의 데이터 뿐만 아니라 나머지 데이터도 삭제 해줘야만 하는 것

데이터의 중복 : 특정 애트리뷰트의 값이 단순하게 여러번 나온다고 해서 값이 중복되는 것은 아님
값이 같아도 다른 의미를 가지는 경우가 있음 ex) 과목번호 : CS310, 이름 이홍근 vs 과목번호 : CS 310, 이름 김순미

*****************기말 예상
ex)이러한 관계형 데이터 모델에서 삽입이상이 발생할 수 있는 .....
답 : ~로 인한 참조무결성 제약조건 위반으로, ~ 삽입이상이 발생할 수 있음

삭제이상 ex) 수강정보 삭제하려다 신상정보까지 묶여 삭제됨
결론 : 수강정보와 신상정보가 묶임으로써 문제 발생, 그룹핑을 잘함으로써 방지가능

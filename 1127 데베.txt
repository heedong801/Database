AVG(SALARY)같은 경우는 새로 생긴 애트리뷰트 이므로 HAVING을 사용해야 한다.
WHERE절에 들어가는 문구는 기존 릴레이션에 있는 애트리뷰트

카테시안 곱
R1 x R2 x R3 ... x Rn
= Select *
   From R1, R2..., Rn

조인

R1 나비 R2 나비 R3... 나비 Rn
= Select *
   From R1...Rn
   Where C1 AND C2 AND ... Cn-1

조인절의 n개의 조인이 이루어질 경우 Where절에 적어도 n-1개의 조인조건이 필요하다.

자체조인
한 테이블에 속하는 같은 애트리뷰트를 검색할 경우, 구별을 위해 새로운 이름을 넣으면 안됨
동일한 애트리뷰트에서 왔을 경우, 어디서 온놈인지 FROM에서 정의하고 구별해주면 됨

중첩질의, 특이한 키워드, 4개이상의 조인, 노멀한 연산

박영권과 같은 직급을 갖는 모든 사원들의 이름과 직급을 검색하라.
중첩질의 없는 답, 선언적인 SQL 문장

SELECT E.EMPNAME, E.TITLE
FROM EMPLOYEE E, EMPLOYEE P
WHERE E.TITLE = P.TITLE AND P.EMPNAME = '박영권'

이퀄 연산자 는 이항 연산자로 좌항 우항이 필요하다.
결과값으로 릴레이션이 나올 경우 IN 연산자로 가능하게 해준다.

ANY 연산자는 매개변수로 들어온 놈 중 하나라도 조건이 만족하면 TRUE
ALL 연산자는 뒤에 집합이 모두 조건에 맞아야 TRUE
